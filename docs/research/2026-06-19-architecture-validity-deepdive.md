## §A CC 自读：快慢思考切换信源依赖（1086 行全文）

源：`~/workspace/raw/01-Wiki/大模型/快慢思考切换信源依赖_enriched.md`

1. **快慢切换 = 五信源条件路由**（`:95-101`），非速度旋钮：意图复杂度分类器[最高] / 端状态[高] / 对话上下文[中] / 负反馈触发词[最高·覆盖性] / 行驶DMS[低·影响展示]。任一信源缺失即降级。
2. **快思考 <200ms / 慢思考 2-5s**（附录术语 `:947-948`）。**慢思考慢是 by-design**，不是缺陷。
3. **🔴 关键反转：生产"快路"不靠大模型**（部署架构 `:671-705`）——
   - 端侧 = 意图复杂度分类器（**BERT，<50ms，<500MB**，`:477-515,692-697`）+ 简单意图 + 端状态查询 + 规则车控执行。
   - **大模型（复杂决策/规划/预测）是【云端】慢思考**（`:681-688,703-704`）。
4. **意图复杂度 4 分类**（`:155`）：简单指令 / 复杂决策规划预测 / 负反馈复思 / 简单计算单指令。
5. **FC 兜底 = 信源(端状态)缺失时的降级方案**（`:949`）；端状态缺失→快思考车控降级为 FC（`:195,961-962`）。
6. 车载 SoC：SA8295P 旗舰 30TOPS = 完整路由+复杂意图分类（`:134`）。端侧基础路由 <100MB、复杂意图分类器 <500MB（BERT 级，非大模型）。
7. 案例：某车企端云融合，快思考 <180ms、慢思考 <3s、意图分类 96.2%（`:987-1001`）。

**对 1.7B 的启示**：router（意图收缩/复杂度判别）是**廉价小分类器/规则**，不是 1.7B；1.7B 只在真做模糊 FC 时调。慢思考慢（2-5s）是常态 → demo 1.7B 模糊理解花几秒是"慢思考正常"，"反应快"由规则快路（<200ms）担。

## §B Subagent L1：多阶任务分类边界 + 架构演进 + 信源矩阵

源：`多阶任务分类边界.md`(1024) + `座舱大模型端到端架构演进.md`(734) + `中文座舱智能体信源矩阵与能力边界.md`(397)

### 多阶任务分类边界（整篇 0 规模断言，最利好 1.7B）
- **判别标准 = 信息完备度，不是延迟/规模**。四问（`:36-39`）：是否需执行 / 槽位目标是否齐 / 有无补全路径 / 否则澄清拒识。
- 精确意图 = `needs_action=true & slot_ready=true` → 短链 FC（`:130`）；模糊 = `slot_ready=false & repair_path=true` → 多阶补全（`:131`）。
- 五判定器最小输入（`:679-685`）：needs_action / slot_ready(+端状态) / subject_type(+说话人) / safety_gate(+驾驶&设备状态) / repair_path(+搜索器)。
- 决策顺序（`:499-506`）：needs_action→**safety_gate**→slot_ready→repair_path→subject_type。**安全门前置**。
- 「多阶不是高级感，而是额外成本」（`:337`），增延迟/扩失败面/损专业感（`:326-330`）。
- 感受映射用**显式 if-else 表**（我好冷→升温/座椅加热，`:651-662`）——规则吃，不耗模型。★1.7B 友好。
- repair_path 可补全条件含"有候选集合可搜索"（`:220`）→ **外部检索强依赖在线，端侧离线砍掉=砍掉慢路主要触发源**。

### 座舱大模型端到端架构演进（规模断言最密集）
- 快/慢精确（`:106-110`）：快思考 **<1s** 明确高频（规则+快意图+直执，**端侧**，低成本）；慢思考 **2-4s** 复杂多步（多阶规划大模型+工具链+COT，**云端**，高成本）。
- 三轨路由（`:358-378`）：置信 >90%→快(端<1s) / 60-90%→澄清 / <60%→慢(云~3s)。驾驶超 500ms 即影响安全感（`:121`）。
- 慢路本质 = Speculative Decoding（`:380-396`）：端 7-14B Draft + 云 70B+ MoE Target，端侧置信低→升云。
- 端侧规模锚点（`:486-491,629-644`）：竞品端侧 12-14B；端侧 MoE 3-14B INT4 跑 SA8295；10B 以下轻量化已可端侧。**预计 2026-2027 端侧 MoE 覆盖 50-60% 云端场景**（`:551`）。
- 成本→规模硬绑（`:133-137`）：低端车（<15万）月均 <1元 → **只能纯端侧规则**；中端混合；高端纯端到端。
- **🔴 简单意图分类器导致 40%+ 误拒**（`:438-441`）——反对纯弱分类器替中枢。
- JSON Schema 约束解码 100% 合规（`:316`，规模无关技术手段）；temp=0.1 + Schema → 100% 格式（`:553-559`）。
- 端到端 ≠ 全塞一个模型（`:667-673`）：量产仍需日志/回退/审计/安全门。

### 中文座舱智能体信源矩阵与能力边界
- 信源对照（`:53-63`）：导航/音乐/新闻/天气/美食 ← 外部在线 API；闲聊知识 ← 云端 LLM；**车控 ← CAN 总线（唯一无外部信源）**。
- 离线硬断言（`:297`）：四个视觉域全部 🔴 需云端 VLM。视觉是可降级旁支（`:301-304`）。
- 承接率天梯（`:81-83`）：车控 22.5% / 导航 35.7% / 中枢 63.6% / 基础 75%。中文整体 ~40%。
- **🔴 车控低承接 22.5% 根因**（`:99-104`）：「28 条后置需求中 22 条归因于 **L2/L3 记忆体系+任务引擎+复杂推理后置至 2027**」「**非车控本身能力不足，而是上层记忆/推理基础设施缺位**」。
- 多意图 ≥93%（双95/三93）vs 讯飞一期 85% 仅并行（`:92`）；拒识四级 vs 二分类（`:93`）；L2 任务记忆后置 26 年底、L3 画像后置 27 年（`:95-96`）。

### L1 综合判断
| | 规模无关（可镜像 1.7B） | 强依赖 8B/云（不可镜像/需重设计） |
|---|---|---|
| | 三层分类入口·五判定器 / 快思考直执 FC / JSON Schema 约束解码 / 感受→车控显式映射 / 安全门是代码 / 单步车控 | 慢思考多阶+COT(云70B) / Draft-Target 推测路由(端云双模型) / 端侧 MoE 红利(1.7B 稠密拿不到) / 多意图≥93%·拒识四级 / L2 任务机·L3 画像 / 多模态VLM / 外部检索 |
- **三句结论**：① 路由/分类/安全/单步 FC 规模无关，1.7B 站得住（多阶分类整篇 0 规模断言=最强证据）；② 真正断裂的是慢思考+推测路由+MoE+多模态+外部检索，但 demo 离线+mock+无多模态**主动砍掉慢路绝大多数触发源**=架构幸运对齐；③ **最大风险**：三篇端侧下限 7-14B/3-14B/10B，**没一篇提 1.7B**，量产端侧比我们高 4-8 倍；硬骨头=多意图/拒识精度/长程记忆，必须 LoRA 窄域补+规则兜底。
- **诚实标注**：三篇均无「1.7B」字样、无 1.7B↔7B 区间 benchmark → 需 spike E3 实测填补。

## §C Subagent L2：MasterAgent + 中枢 + 编排（算法级）

源：`MasterAgent车控智能体.md`(2069) + `MasterAgent与中枢智能体调用链及Agentic-Skill分工.md` + `座舱智能体编排与多Agent架构2026Q2.md` + `中枢智能体架构与需求.md`

### MasterAgent（执行层，非中枢）
- 定位：FC 之上的高阶车控，处理「我冷」感受类，非「开空调26度」明确指令（`MasterAgent:38,633`）。唯一同时支持「多阶推理+多意图(一句多执行)+端状态依赖(必须)」（`:562-572`）。
- **🔴 "多阶" 算法本质 = 单模型 ReAct 循环（思考-行动-观察），不是多 Agent**（`:778-780`）。车载 ≤2-3 轮、每轮 400-600ms、总 ≤2500ms（`:784-786,1319`）。两工具：`query_vehicle_state`（生成计划前必调）+ `execute_vehicle_control`（`:821-869`）。
- 三态端状态推理：满足/不满足/仅感知（`:721-724`，**确定性规则**）；二级推荐（一级到极值触发，`:316-325`）。
- **模型规模：MasterAgent = 8B 云端主力**（`:635,689`），LLM 推理 800-1200ms（`:1442`）。P2 才提端侧 1-3B 接单垂域简单（`:1462`）。弱网→退端侧规则引擎/FC 直执（`:1466-1480`）。
- 多意图「一句多执行」指**同垂域多执行项**（空调+座椅+方向盘），跨垂域多意图列中长期（`:566,652,1302`）。

### 调用链 + Agentic-Skill 分工
- **三层**（`调用链:98-105`）：认知层(基础智能体,对话/Persona/L1-L3记忆) / 调度层(**中枢**,路由≥93%/仲裁/拒识,幕后) / 执行层(**MasterAgent**,端状态多阶车控)。「MasterAgent 不是中枢，是中枢路由的专域智能体之一」。
- Quick/Smart（`:117-149`）：Quick=端侧 Mini LLM **3-7B**,<200ms,简单/已知/紧急；Smart=云端中枢,2-4s,复杂/跨域/实时。
- **🔴 三种 Agentic Skill**（`:152-256,481`）：
  - ReAct（思考-行动-观察）：模糊探索/多轮澄清；**端侧适配 ❌**（Token 线性增长，端侧 SLM 上下文 4K-8K **超 5 轮截断**，`:310-314`）。
  - **Plan-and-Execute：Planner=大模型 70B+ / Executor=小模型 3-7B**，降本 90%（`:181-188,336-339`）；**端侧适配 ✅（Executor 可用 SLM）**。
  - Multi-Agent DAG：中枢作 Orchestrator 并行；**Joiner 需大模型 ⚠️**（`:205-235,417`）。
- **🔴 算法级规模分工**：编排/规划（Planner、Joiner、Orchestrator 仲裁）**强依赖大模型**；执行（Executor）可小模型。「编排脑需大模型，执行手可小」。
- 端侧 SLM FC 标杆：Phi-3 mini **3.8B FC 92%**（Berkeley，`:558-567`）。五级 FC 延迟：L1 车控 <100ms（规则+SLM FC）（`:603-611`）。

### 编排与多 Agent 架构 2026Q2
- **异构三层模型路由**（`2026Q2:156-160`）：编排层 Frontier **100B+** 云（复杂推理/多步规划/编排决策 2-4s）/ 标准层 7-13B 云边 / 执行层 **SLM 2-4B** 端 NPU（意图分类/简单 FC/车控 <0.3s）。
- 「**端侧 SLM 是实时性保障**，车控 <200ms 端侧 NPU 是唯一稳定路径」（`:27`）。
- **🔴 车控 FC 专项：1B 量化（Gemma3 1B/Llama3.2 1B）→ CPU 160+ TPS、首包 <200ms ✅**（`:719`）；复杂 NLU 要 3-4B（`:720`）；多语种 7B；**禁 Q4 以下（幻觉 3x）**。
- ReAct 工具名幻觉 → **工具路由下沉代码层、名称编译时固定**（`:760,368`）；retry 浪费 90.8%。
- DST 信念状态 b_t(d,s,v)，置信 ≥0.8 确认否则澄清（`:189-222`）；VehicleMemBench：强模型直接指令好、动态偏好退化（`:668-677`）。

### 中枢智能体架构与需求
- 中枢 8 大能力（`中枢:85-188`）：复杂语义(复合/条件/排除/指代≥90-95%) / **多意图 2-5 拆解+优先级(双95/三93/四+90)** / 否定≥95% / 对话管理(记忆≥3轮/跨域继承/中断恢复) / 纠错 / 拒识(误拒≤5%/安全拒识≥98%) / 路由(≥95%/≤200ms/15+垂域) / 输出仲裁(安全>任务>置信>偏好>可行)。
- 「中枢不是再包一层大模型，是**系统级决策门**」（`:307`）；4 接口 intent_parse/route_select/candidate_merge/safe_finalize（`:318-323`）；「中枢真正难的是**结构化拆解后仍保持一致**」（`:365`）；统一中间表示 intent_cluster/sub_intents/constraints/execution_mode/risk_level/memory_need（`:372-381`）。
- 部署：云端中枢+端侧调度（`:69`）；三层记忆 L1对话/L2任务/L3画像（`:226-230`），安全/拒识/合规不可被记忆影响（`:234`）。

### L2 综合判断
- **1.7B 甜区 = "执行手"（FC+单垂域端状态推理），非"编排脑"（多意图拆解+多步规划+跨域仲裁）。文档反复实证编排/规划要 70B-100B+，执行才 1-4B。**
- 1.7B 能演**降级 MasterAgent**（ReAct 砍/三态用规则/限同垂域 2-3 轮/工具名编译固定）；**不能真做中枢**（多意图 2-5 拆解+跨域仲裁需大模型）。
- ⭐ demo 取舍：保留「单 1.7B+三态规则+同垂域多执行项」，砍「真多 Agent 编排+Plan-Execute Planner+跨域仲裁」；同垂域多意图保留（规则触发多执行），跨域多意图用**规则切分+串行 FC**（不做真 Planner/DAG）；多步规划 demo 不做（Planner 需 70B+）。
- **诚实标注**：无任何文档提「1.7B 跑 MasterAgent/中枢」；1.7B 落在「1B 车控 FC」和「3-4B 复杂 NLU」之间，文档无直接基准 → spike E3 验。

## §D Subagent L3：客户生产方案 + 锁域 + 星火2.0（协议/算法级）

源：`02-Raw/客户反馈/【客户方案】奇瑞...解决方案1.0.md` + `交互大模型交付手册.md` + `星火方案2.0/技术架构与竞争力.md` + `端到端架构对比.md` + `智能体锁域机制与拒识策略.md`（红线：客户名→「某车厂」，未抄报价/密钥/PII）

### 生产架构精确形态
- **路线 = "语义端到端增强的混合级联"**（`端到端对比:23,31-42`）：可观测级联骨架（ASR/VAD/协议/**策略门**/执行）保安全下限 + 大模型嵌**语义理解/工具参数/回复/多阶推理**保上限。大模型放语义/工具参数/回复/多阶（`:49`）。
- **模型家族（精确，§28）**（`技术架构:915-921`）：**2B 端侧**(快NLU/简单意图) / **2.6B**(拒识判别) / **7B·30B-A3 端云**(语义增强/中推理) / **14B 云**(GUI Agent) / **130B 云**(复杂规划/长任务)。**五篇无「8B」字样**；MasterAgent 8B 见 §C（`MasterAgent:635`）——磊哥说的"8B 中枢"对应 MasterAgent 执行体规格，重推理主力是 **130B 云**。
- 端云划分（`技术架构:444-465`）：简单/离线/隐私→端侧 SLM 2B/7B/30B-A3；复杂/长任务→云端 14B~130B。三维路由：复杂度/连通性/隐私。端侧首响 40ms（依赖**车规 NPU**，`:749`，缺失退化纯云 4-5s）。

### "传统语义和 FC 都走快速方案" = 3 条快路（纠正 FC 理解）
- ① **aqua 干预层**：高置信意图快速直达技能（`锁域:110-138`）；
- ② **文法快修**：规整/拒识/内置落域/知识分类可文法干预，**不必每月模型迭代**（`交付手册:49,876`）；
- ③ **端侧 SLM 闭环**：简单原子车控 40ms（`技术架构:444-450`）。
- **🔴 关键**：FC 在量产是**云端 14B+ 用 constrained decoding 输出 JSON = 慢路**；**只有端侧 SLM 才是 FC 的快路实现**。→「FC 走快路」指**端侧小模型 FC**，恰好利好 1.7B demo。

### FC 算法（demo 最该抓，规模无关）
- 受限解码（constrained decoding）：token 采样被 JSON Schema 约束，依赖 CFG/FSM（`技术架构:301`）；enum 白名单（97 车控原子能力）把幻觉压近 0（`:304,327-337`）。FC ≤2400ms；L1≥99%/L2≥85%/L3≥80%，误召≤3%（`:171`）。
- **FC 脆弱性**：JSON all-or-nothing，弱网截断整条作废；兜底=端侧闭环优先+约束解码+失败重试（`:339-340`）。

### 拒识（独立 2.6B 判别式）
- 判别任务不需生成，不必 14B+；2.6B 端/云可部署、比 BERT 泛化强=「能力够用+成本最优甜点」，独立迭代/灰度（`技术架构:393`）。代价函数 `Cost=w_FRR·FRR+w_FAR·FAR`（单人调高 w_FRR 少拒/多人调高 w_FAR 多拒，`:389-391`）。
- 拒识依据：上下文关联度/完整性/通顺度/可响应性/黑白名单/技能落域；召回 97%（`交付手册:824`）；端收 `cbm_denial` type=0（`:818-819`）；对外正确率≥90%、误拒≤5%（`:836-838`）。

### 落域 + 锁域（纯逻辑，端侧可复制）
- 落域 = **aqua（快路高置信）+ 内置落域（模型二分类）并行**（`交付手册:256`）；内置落域字段 `cbm_intent_domain`，对外准确率≥95%（`:252,260`）。

### W3：语义+规则混合方法
**结论：磊哥"规则快路 L1 + 小模型慢路"=Snips/Dialogflow 跑多年的成熟范式（"先确定性、不中再概率"），且受限解码对 1.7B 几乎必需。**

- **Dialogflow**：rule grammar + ML 并行择优，阈值门控；显式建议"样本少/模板多→hybrid，样本多→ML-only"（demo 早期样本少→规则占比应更高）。
- **Rasa**：rules.yml 确定性 + ML policy + FallbackClassifier(低置信/歧义→规则接管)；**警告"置信度不是真概率，只是与训练数据相似度"→阈值用实测 sweep 校准**。
- **LUIS→CLU 去规则化=paper-tiger**：只在云端大模型+海量数据成立；1.7B 端侧反向，规则快路要留。
- **受限解码（1.7B FC 必需品）**：GBNF(llama.cpp 原生)/XGrammar(<40μs/token,支持递归,vLLM/SGLang/MLC 默认)/Outlines(FSM)。**XGrammar-2 实证 1B：6.07%→32.84% correct、22%→100% schema；3B+约束反超 70B 裸跑**。**caveat：保格式不保语义（"valid-but-wrong"，smollm3-3b 格式 36→86% 但准确率没动）→ 语义靠 LoRA+规则+DemoGuard**。
- **router 拍板**：**用规则关键词+embedding 语义路由（22MB CPU ms 级，car-sales chatbot 92-96% precision 零训练），绝不用 LLM 当 router**（"routing is classification not generation"）。vLLM Semantic Router v0.1(2026)=Keyword Signals(regex)+Embedding Signals 多信号 AND/OR 决策。
