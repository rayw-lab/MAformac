# UIUE R4 Pre-Grill Classification Table

Purpose: classify the existing R4 v2 C01-C50 grill matrix before human review. This is a routing table, not a new grill list and not an implementation verdict.

Source matrix: `docs/loop-competition/2026-06-28-uiue-r4-bridge-grill-matrix/final-grill-matrix-v2.md`. Mechanical source row count: 50.

## Bucket Contract

- `bridge schema`: Defines machine-readable snapshot/result contract or SSOT hygiene; must not absorb visual-only behavior.
- `visual policy`: Defines UI behavior, zone ownership, gesture arbitration, choreography, or layout policy outside schema.
- `evidence checklist`: Defines proof-class, receipt, screenshot, UI tree, timing, or harness requirements.
- `mainline co-author`: Requires mainline/runtime roadmap co-authorship or authority reconciliation before R4 exit.
- `R5 deferred`: R4 may define seam/non-claim only; runtime, voice, model, or golden readiness belongs after R4.
- `user decision`: Needs product/operator decision before it can become visual policy or implementation work.

C50 is the governance gate: no UI detail may be inserted into bridge schema until classified as `bridge schema`, `visual policy`, or `evidence checklist`; some items instead route to `mainline co-author`, `R5 deferred`, or `user decision`.
