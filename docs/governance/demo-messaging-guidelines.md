## 1. 禁说清单

下列词汇在任何对外材料中**一律不得出现**，包括口播、PPT、字幕、文档、UI 文案、issue、PR 描述、commit message（对外可见分支）。

| 禁说词 | 为什么禁 | 替代表达 |
|---|---|---|
| 端侧大模型 | App 仍未接任何模型；有限评审目录不能冒充模型能力 | 本地规则快路径（12 个评审入口） |
| Qwen | 具体模型名，未集成；点名即构成事实错误 | （不点名；若被追问直说「未接模型」） |
| LoRA | 微调术语，未做；暗示训练/定制能力 | （无对应能力，不替代，直接说「未做」） |
| 语音交互 | 未实现语音通道 | 文本输入验证语义闭环 |
| 自然语言理解 / NLU | 未接 NLU，只有有限评审目录与参数模板 | 有限文本目录验证 |
| 多意图 | 当前多意图未准入；说多意图即夸大 | 单意图评审目录 |
| 泛化 | 未做泛化；目录外输入 fail-closed | 评审话术覆盖 |
| 记忆 | 无用户/会话状态持久化 | mock state store |
| 功能安全 | 「功能安全」在车规域是受规管术语（ISO 26262），不可口语化使用 | （不使用；明确「未接真实安全判定」） |
| ISO 26262 | 未过任何车规认证；引用标准名即暗示合规 | （不引用；直说「未通过车规认证」） |
| ASIL | ASIL 等级术语，未评定，不可借用 | （不使用） |
| 已验收 | 验收体系建设中，actionDemoProven = 0/120，无任何条目标记通过 | 验收体系建设中 |
| V-PASS | 不存在 V-PASS 这一已达成的里程碑；仅为内部流程代号 | （不对外使用） |
| 生产就绪 / Production Ready | Phase 0，离量产不可达 | 演示原型 |

> 禁说词不是「换个说法」的同义词替换题——其中 7 个（Qwen / LoRA / 语音 / NLU / 多意图 / 泛化 / 记忆）对应当前**未实现或未准入的能力**；直接说「未接 / 未准入」才是正确答案。

## 2. 可说清单

下列词汇是经过事实锚验证的安全表达；使用时必须严格按「适用场景」限定，**不得外溢**。

| 可说词 | 适用场景 |
|---|---|
| 规则引擎快路径 | 描述当前 12 个评审入口的路由机制；只在被问到「怎么实现的」时使用 |
| 有限文本目录验证 | 描述「输入 → 目录匹配 → 命中/拒绝 → 状态读回」；不得暗示自然语言理解 |
| mock state store | 描述 demo 中 UI 状态的来源是本地 store，不是真实车机回读 |
| 目录外 fail-closed | 描述未评审文本不执行、不改状态；不得包装成安全判断 |
| 演示原型 | 描述当前产物的性质（prototype / demo），任何场合都可以用 |
| 验收体系建设中 | 回答「做完了吗 / 过验收了吗」的标准答案；附带事实锚 |
| 模拟效果 | UI 水印与口播，用于标识非量产性质（见 safety-disclosure-and-qa.md） |

## 3. 防问倒标准答（5 条）

以下是被问到高风险问题时的**标准应答模板**。背熟、照念、不即兴发挥。

### Q1：「你们这是大模型做的吗？端侧跑的什么模型？」
> 不是。当前 App 没有接入任何模型；车控路径走有限评审目录，包含 12 个入口与空调调温参数模板，目录外 fail-closed。模型接入是下一阶段的产品工作，今天不冒充。

### Q2：「能识别多少种说法？换个说法还行不行？」
> 只支持演示脚本和 manifest 里评审过的说法；**不承诺同义改写**。当前没有 NLU 与泛化，目录之外应拒识，不能把偶然命中解释成「听懂了」。

### Q3：「现在过验收了吗？离量产还有多远？」
> 都没有。当前 actionDemoProven = 0/120，验收体系还在建设中，一条 V-PASS 都没有。今天给各位看的是**演示原型**，不是量产形态。

### Q4：「这个安全机制是真的吗？在车上能跑吗？」
> 今天**不演任何安全拦截**。画面里的状态来自 mock state store，不是真车机信号；真实档位、车速、CAN 与量产功能安全均未接入。

### Q5：「（车停稳了）现在能开窗了吗？」
> 脚本里有一条精确开窗话术，可以演示 mock store 的执行和读回；但停稳/行驶安全判定不在当前产品边界，我不现场即兴演这类判断。

## 4. 事实锚（每次对外沟通必带）

> 产品当前正向目录=12 个评审入口（空调调温是 18–32 整数边界模板，其余为精确话术）/ App 仍未接模型 / 多意图未准入 / actionDemoProven=0/120

- **口播场合**：演示开场 45 秒内必须完整念出一遍事实锚与 mock 声明（见 safety-disclosure-and-qa.md §1.2）。
- **文档场合**：任何对外文档（方案、PRD、宣讲材料）首页或第一段必须包含事实锚原文。
- **UI 场合**：演示界面水印必须持续可见，文案见 safety-disclosure-and-qa.md §1.3。
- **验收唯一标准**：磊哥亲灌句 + 看每周录像。三腿凳（人工走查 / 自动化回归 / 防忘 checklist）是辅助信号，**不是**验收通过的依据；任何汇报不得用三腿凳结果暗示「已通过」。
- **进度汇报禁令**：禁百分比进度（「完成 80%」「进度 60%」一律不得出现）；进度只能用 `proven/total` 分子分母形式（当前 = 0/120）。

## 5. 违规示例（常见错误 → 正确话术对照）

| 场景 | 错误话术（禁） | 正确话术（可） |
|---|---|---|
| 开场介绍 | 「我们在端侧跑了一个大模型…」 | 「今天的演示没有接入模型，走有限评审目录，共 12 个入口。」 |
| 被问实现原理 | 「用 Qwen 做了 LoRA 微调…」 | 「没有用任何模型；是本地目录匹配与契约执行，目录外 fail-closed。」 |
| 被问语音 | 「支持语音控制…」 | 「当前只支持文本输入，真实 ASR 未实现。」 |
| 被问泛化 | 「能理解各种自然说法…」 | 「只执行脚本与 manifest 里的评审话术，不承诺同义改写。」 |
| 被问记忆/个性化 | 「它会记住你的习惯…」 | 「没有用户状态持久化；你看到的偏好是 mock 数据。」 |
| 被问安全 | 「符合功能安全 / 过了 ASIL…」 | 「当前不演安全拦截；未接真车信号，也未通过 ISO 26262。」 |
| 被问验收 | 「已经验收通过了 / V-PASS 了 / 生产就绪」 | 「验收体系建设中，当前 actionDemoProven=0/120，一条都没通过。」 |
| 汇报进度 | 「完成 80% 了 / 差不多了」 | 「当前 0/120；验收标准是磊哥亲灌句 + 每周录像，不以百分比计。」 |
| 被问停稳场景 | 「停稳就能开，你看——」（现场试） | 「开窗评审句只证明 mock store 执行；停稳/行驶判定未接，不即兴演。」 |
| 被问开窗/其他功能 | 「什么车控都能做…」 | 「只演 12 个评审入口；目录外不执行，不能外推到其他车控。」 |
| 被追问换说法 | （换个说法试了试，碰巧命中）「你看也能识别」 | 「不承诺同义改写；我不现场即兴试，避免制造泛化错觉。」 |
